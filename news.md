# news.md
제작시스템 기술 명세서이다.

### 서버
##기술명세서
- NodeJS를 사용한다.
- MVC Pattern으로 개발한다.
- DB는 SQLite를 사용하고 상세 스펙은 기사 DB 명세서를 따른다.
##기사 함수
- 기사를 입력/수정/조회 할 수 있는 함수를 생성한다.
- 기사를 조회하는 함수에서는 배부시간을 조건으로 기사를 조회할 수 있다.
- 기사를 조회하는 함수에서는 작성시간을 조건으로 기사를 조회할 수 있다.
- 기사를 조회하는 함수에서는 기사 아이디를 조건으로 기사를 조회할 수 있다.
- 기사를 조회하는 함수에서는 작성자를 조건으로 기사를 조회할 수 있다.
- 기사를 조회하는 함수에서는 송고자를 조건으로 기사를 조회할 수 있다.
- 기사를 수정하는 함수에서는 기사 아이디를 조건으로 기사 상태값을 수정할 수 있다.
- 기사를 삭제하는 함수에서는 기사 아이디를 조건으로 기사 상태값을 수정할 수 있다.
##사용자 함수
- 사용자를 입력/수정/삭제/조회 할 수 있는 함수를 생성한다.
- 사용자는 조회하는 함수에서 아이디와 비밀번호를 조건으로 전달된 값이 DB에 있는 정보와 같으면 로그인 할 수 있다.
- 로그인 후에는 기사 작성 기능과 기사 목록기능이 있는 페이지로 이동한다.

##워크플로우
- 클라이언트에서 요청받은 데이터를 사용하여 기사 생애주기를 거쳐 DB에 적용된다.

# 공통 조건
- 모든 페이지에는 우측 상단에 로그인한 사용자의 정보를 보여준다.

# 기사 작성페이지
- 기사 작성페이지는 에디터와 우측에 기사 메타데이터를 입력하는 레이아웃으로 구분한다.
- 기사 메타데이터를 입력하는 레이아웃에서 탭이 4개가 있는데 공통정보/이미지/영상/글기사가 있다.
- 4개 탭 위에 송고 보류 버튼이 있다.
- 공통정보에는 작성자, 공동작성, 내용, 지역, 속성, 키워드, 내부코멘트, 외부코멘트, 첨부파일, 자료파일을 입력받고, 엠바고, 2차 엠바고 시간을 입력받는다.
- 이미지와 영상, 글기사는 본문에 임베딩으로 사용할 데이터를 조회한다.
- 이미지와 영상은 유투브를 검색하고 실패시 구글 검색을 사용한다.
- 글기사는 내부 기사 제목, 본문내용을 검색한다.

# 기사 조회페이지
- 기사 조회페이지는 실시간이다.
- 실시간 상태를 보여주는 상태바 우측 상단에 있다.
- 메뉴가 4개 있는데, 부서별 작성, 부서별 송고, 개인별 수정, 데스크 미송고 메뉴가 있다.
- 부서별 작성페이지는 해당 부서에서 작성된 기사만 조회한다.
- 부서별 송고페이지는 부서별로 송고할 수 있는 드롭다운 메뉴와 조회버튼을 통해 조회할 수 있다.
- 개인별 수정페이지는 개인이 작성한 기사만 조회 할 수 있다.
- 데스크 미송고페이지는 부서에서 작성된 기사와 RDS기사만 조회한다.

##워크플로우
- 로그인 페이지에서 아이디와 암호를 서버에 전달해 인증이 성공하면 기사 작성페이지로 넘어간다.
- 로그인 페이지에서 아이디와 암호를 서버에 전달해 실패하면 실패한 이유와 함께 ALERT 메시지를 띄운다.
- 로그인 성공한 사용자는 송고 버튼을 눌러 기사DTO를 서버에 전달해 기사를 DB에 적용한다.

## 기사 DB 명세서
- 테이블은 Article, Contents, User 3개를 만든다.
- 3개의 테이블에 속성을 추가하는데 Article에는 ArticleVO.md, Contents에는 ContentsVO.md, User에는 UserVO.md를 읽어 필드명대로 추가한다.

## SQLite Store Procedure 명세서
- 기사 아이디를 생성해주는 SP를 만든다.
- 기사이이디 규칙은 'AKR' + YYYYMMDD + 난수 9자리로 생성한다.

## API 명세서

## 사용자 권한
- 기자 리포터 R, 국기사 데스크 D, 관리자 Z

## 기사 제어 권한
- 기사는 사용자권한 R, D, Z 편집 할 수 있다.
- 기사의 상태값이 DPS일 때는 D 권한 사용자만 고침/포털고침 메뉴를 사용할 수 있다.

## 기사 생애주기
- 기사가 최초 작성시에는 상태값이 RDS이다.
- 권한 R 사용자가 RDS 기사를 송고시에는 RDS가 된다.
- 권한 R 사용자가 RDS 기사를 보류시에는 RRH가 된다.
- 권한 R 사용자가 RDS 기사를 KILL 하면 RRK가 된다.
- 권한 D 사용자가 RDS 기사를 송고시에는 DPS가 된다.
- 권한 D 사용자가 RDS 기사를 보류시에는 DDH가 된다.
- 권한 D 사용자가 RDS 기사를 KILL 시에는 DDK가 된다.
