# Product — 기사 작성기 (제작 시스템)

## 프로젝트 개요
언론사용 **기사 제작 시스템(CMS)** 이다. 기자·데스크·관리자가 기사를 작성·수정·송고·조회하고, 기사 생애주기(상태값)를 관리한다. 전체 제품은 **제작 / 수집 / 배부** 3개 시스템으로 구성되며, **현재 범위는 제작 시스템만** 구현한다.

기준 명세: `news.md`, `ArticleVO.md`, `ContentsVO.md`, `UserVO.md`.

## 대상 사용자
| 권한 코드 | 역할 | 설명 |
|-----------|------|------|
| R | 기자(리포터) | 기사 작성·송고·보류·KILL |
| D | 국기사(데스크) | 기사 편집, DPS 상태 기사 고침/포털고침 |
| Z | 관리자 | 기사 편집 가능 |

## 핵심 기능
- **기사 함수**: 입력 / 수정 / 조회 / (상태값 기반) 삭제
  - 조회 조건: 배부시간, 작성시간, 기사 아이디, 작성자, 송고자
  - 수정·삭제: 기사 아이디 기준으로 상태값 변경
- **사용자 함수**: 입력 / 수정 / 삭제 / 조회, 아이디·비밀번호 기반 로그인
- **기사 작성 페이지**: 에디터 + 우측 메타데이터 입력(탭 4개: 공통정보 / 이미지 / 영상 / 글기사), 송고·보류 버튼
  - 이미지·영상: 유튜브 검색 → 실패 시 구글 검색
  - 글기사: 내부 기사 제목·본문 검색
- **기사 조회 페이지**: 실시간, 상태바 표시, 메뉴 4개(부서별 작성 / 부서별 송고 / 개인별 수정 / 데스크 미송고)
- **기사 ID 생성 SP**: `AKR + YYYYMMDD + 난수 9자리`

## 기사 생애주기 (상태값)
- 최초 작성: **RDS**
- R 권한: 송고→RDS, 보류→RRH, KILL→RRK
- D 권한: 송고→DPS, 보류→DDH, KILL→DDK
- Z 권한: 송고→DPS, 보류→DDH, KILL→DDK (D 권한과 동일한 전이 규칙, plan.md D-6 / SPEC-NEWS-REVISE-001 AC-Z-LIFECYCLE-1)
- 상태값 **DPS**: D 권한 사용자만 고침/포털고침 가능

## 워크플로우
1. 로그인 페이지에서 아이디·암호를 서버에 전달 → 인증 성공 시 기사 작성 페이지로 이동
2. 로그인 사용자가 송고 버튼 → 기사 DTO를 서버에 전달 → 기사 생애주기를 거쳐 DB에 반영

## 범위 (Scope)
- **In**: 제작 시스템(서버 + 클라이언트), 기사·사용자 CRUD, 로그인, 생애주기, 작성·조회 페이지
- **Out (향후)**: 수집 시스템, 배부 시스템

## 제약 / 규칙
- 모든 텍스트는 UTF-8로 작성·저장한다.
- DB에 있는 내용은 삭제하지 않는다(상태값 변경으로 처리).
- 모든 페이지 우측 상단에 로그인 사용자 정보를 표시한다.
