# 기사 작성기 — 송고 상태 전이 및 list.do 메뉴 필터

출처: `D:\agents\tech_day\news.md` (source-of-truth) + 스킬 `moai-domain-news-editor/SKILL.md`

## (1) R 권한 사용자가 RDS 기사를 송고하면 → **RDS (유지)**

R 권한(기자 리포터, Reporter)이 RDS 상태 기사를 송고해도 상태값은 **그대로 RDS**로 유지된다. DPS로 전환되지 않는다.

- **근거**: news.md L154 — "권한 R 사용자가 RDS 기사를 송고시에는 RDS가 된다."
- 의미: R의 송고는 데스크(D) 검수 대기 단계일 뿐, 배부 대상으로 전환되지 않는다.
- 스킬 근거: SKILL.md §1.3, §2.2 생애주기 전이 매트릭스(`RDS | R | 송고 → RDS`), §2.7 도메인 오해 2번("R 권한 송고 후 상태는 DPS"는 오해).

## (2) D 권한 사용자가 RDS 기사를 송고하면 → **DPS**

D 권한(국기사 데스크, Desk)이 RDS 상태 기사를 송고하면 상태값이 **DPS**가 되며, 이때부터 **배부(distribution) 대상**으로 전환된다.

- **근거**: news.md L157 — "권한 D 사용자가 RDS 기사를 송고시에는 DPS가 된다."
- 스킬 근거: SKILL.md §1.3(`DPS` = D 권한이 송고한 RDS 기사 = 배부 대상), §2.2(`RDS | D | 송고 → DPS`, "배부 대상으로 전환").
- DPS 기사는 list.do의 **부서별 송고 메뉴**에서 조회되며, 이 상태 + D 권한에서만 고침/포털고침이 활성화된다.

### 참고: 전체 RDS 송고/보류/KILL 전이 매트릭스 (news.md L153-159)

| 초기 상태 | 권한 | 액션 | 결과 상태 |
|-----------|------|------|-----------|
| RDS | R | 송고 | **RDS** (유지) |
| RDS | R | 보류 | RRH |
| RDS | R | KILL | RRK |
| RDS | D | 송고 | **DPS** |
| RDS | D | 보류 | DDH |

### 질문 대상 두 메뉴 정리

- **부서별 송고 메뉴**: 상태값이 **DPS인 기사만** 조회한다 (부서 드롭다운 + 조회버튼). 즉 D 권한이 송고를 완료해 배부 대상이 된 기사들이 모이는 곳이다.
  - 근거: news.md L74-75 — "부서별 송고페이지는 부서별로 송고할 수 있는 드롭다운 메뉴와 조회버튼을 통해 조회… DPS기사만 조회 할 수 있다."

- **개인별 수정 메뉴**: 로그인한 계정이 직접 작성한 기사 중 상태값이 **RDS, RRK인 기사만** 조회한다.
  - 근거: news.md L76 — "개인별 수정페이지는 로그인한 계정이 작성한 기사 중 상태값이 RDS, RRK인 기사만 조회 할 수 있다."
  - 해석: 아직 송고 전 초안(RDS)과 본인이 KILL 처리한 기사(RRK)를 본인이 다시 수정할 수 있도록 모아주는 메뉴다.
