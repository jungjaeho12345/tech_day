## (1) Z 권한 사용자에게 보이는 버튼

Z = **관리자(Admin)** 권한이다. 기사 작성 페이지(writer.do)에서 **RDS 상태 기사**에 한정해, Z 권한 사용자에게는 다음 **3개 버튼이 모두 보이고 사용 가능**하다.

| 버튼 | Z 권한 + RDS |
|------|-------------|
| 송고 | 보이고 사용가능 |
| 보류 | 보이고 사용가능 |
| KILL | 보이고 사용가능 |

핵심 구분 포인트:
- D 권한은 **KILL 버튼이 비표시**이지만, Z 권한은 R 권한처럼 **3개 버튼이 전부 노출**된다. (송고·보류·KILL 모두 가시·사용가능)
- RDS **외** 상태(RRH/RRK/DPS/DDH/DDK 등)에서는 권한과 무관하게 3개 버튼 모두 **비표시**된다.
- **주의(흔한 오해)**: "Z 권한 = 모든 버튼"은 오해다. 이 가시성 규칙은 *작성 페이지의 송고/보류/KILL 3 버튼 한정*이다. 고침/포털고침/재송/삭제요청 등 일반 액션은 별도 권한 매트릭스(SPEC-NEWS-REVISE-001)를 따른다. Z 권한의 작성·편집 권한 범위도 "기사 작성 및 데스크 미송고 편집" 한정이다.

## (2) 본문에 `(끝)` 없이 송고 버튼을 누르면

**송고는 차단된다.**

송고 가드 규칙(SPEC-NEWS-REVISE-005, news.md L66):
- 송고는 **본문에 `(끝)` 마커가 있어야만** 진행된다.
- `(끝)`가 없으면 **ALERT를 띄우고 송고를 차단**한다. 즉 `articleUpdate` 호출로 이어지지 않고 상태값 전이(RDS→DPS 등)도 일어나지 않는다.
- 보류/KILL은 이 가드의 영향을 받지 않는다 — **`(끝)` 없이도 진행**된다.

`(끝)` 마커는 `Alt+Y`로 1회 삽입되며(이미 있으면 재삽입 안 함), embeds 뒤 **최종 블록**으로 배치된다(시각 순서: 본문 → embeds → `(끝)`). `getBodyText()`는 trailing `(끝)`으로 끝나므로 송고 가드를 통과한다.

추가로, `(끝)`가 있어 가드를 통과하더라도 **제목이 없으면 송고/보류 실패 ALERT**가 별도로 발생한다(news.md L105). 또한 송고 버튼은 먼저 '송고하시겠습니까?' **확인창**을 띄우고 확인 시에만 진행하며, 성공 시 버튼 아래 상태 메시지는 표시하지 않고 작성 페이지를 초기화한다.
