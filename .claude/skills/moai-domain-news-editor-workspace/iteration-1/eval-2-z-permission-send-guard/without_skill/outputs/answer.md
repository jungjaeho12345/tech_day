## (1) Z 권한 사용자에게 보이는 버튼

Z 권한 사용자에게는 **송고 / 보류 / KILL 3개 버튼이 모두 보인다.**

단, 이 3개 버튼은 **편집 중인 기사의 상태값이 RDS일 때만** 노출된다. (`isRds = ctrl.status === 'RDS'`)

코드 근거 (`WritePage.jsx`):
- 송고·보류: `(user.role === 'R' || user.role === 'D' || user.role === 'Z') && isRds`
- KILL: `(user.role === 'R' || user.role === 'Z') && isRds`

| 권한 | 송고 | 보류 | KILL | 조건 |
|------|------|------|------|------|
| R    | O    | O    | O    | RDS |
| D    | O    | O    | X    | RDS |
| **Z**| **O**| **O**| **O**| RDS |

즉 Z는 R과 동일하게 송고/보류/KILL 전부를 사용할 수 있으며(news.md "Z권한은 송고/보류/KILL 버튼이 보이고 사용할 수 있다", 그리고 "Z권한도 R/D와 동일한 RDS gate 적용" 결정 잠금 REQ-AUTH-Z-BUTTONS), 상태값이 RDS가 아닌 기사를 편집 진입한 경우에는 어떤 권한이든 이 액션 버튼이 전혀 노출되지 않는다.

> 참고: 이는 "기사 작성 페이지 내 액션 버튼"에 한정한 답이다. 페이지에는 그 밖에 4개 탭(공통정보/이미지/영상/글기사), 탭별 검색·삽입 버튼, 로그아웃 등이 권한과 무관하게 공통으로 존재한다.

## (2) 본문에 "(끝)" 없이 송고 버튼을 누르면

**송고가 차단된다.** 동작 순서는 다음과 같다.

1. 송고 버튼 클릭 → 먼저 `window.confirm('송고하시겠습니까?')` 확인창이 뜬다. 취소하면 아무 일도 일어나지 않는다.
2. 확인을 누르면 `ctrl.send()` → `submitAction('send')`로 진입한다.
3. 가드 순서:
   - (lockError가 있으면 모든 전송 차단)
   - 제목 가드: 제목(에디터 첫 줄)이 비어 있으면 인라인 에러 "제목이 없어 송고/보류할 수 없습니다."를 표시하고 중단.
   - **(끝) 가드**: `if (action === 'send' && !hasEndMarker(adapter.getBodyText()))` 가 참이면
     `window.alert('본문에 (끝) 표시가 없어 송고할 수 없습니다.')` **ALERT를 띄우고 즉시 `return`** 한다.
4. 이 가드는 `saveArticle`(저장)과 `applyAction`(상태 전이) **호출 이전**에 위치하므로, 저장도 서버 전송도 전혀 일어나지 않는다. 기사 DB는 변하지 않고, 페이지 초기화도 발생하지 않으며, 작성 중 내용은 그대로 유지된다.

`hasEndMarker` 판정 기준 (`editorContent.js`):
- 본문 텍스트를 `trimEnd()` 한 결과가 `"(끝)"`(`END_MARKER`)로 끝나면 통과.
- 즉 본문 **맨 끝**에 "(끝)"이 있어야 하며, 임베드(이미지/영상/글기사)는 텍스트에 기여하지 않으므로 "(끝)"이 최종 블록이면 임베드가 뒤에 있어도 통과한다(레거시 `"\n (끝)"` 형태도 인정).
- "(끝)"은 보통 **Alt+Y** 단축키로 골드색 최종 블록으로 삽입한다.

중요 구분:
- **(끝) 요구는 송고(send)에만 적용된다.** 보류(hold)·KILL은 "(끝)" 없이도 진행된다 (news.md: "보류/KILL은 (끝) 없이도 진행된다").

## (3) 검증 체크리스트 (5항목)

| # | 시나리오 (Given / When) | 기대 결과 (Then) |
|---|--------------------------|------------------|
| 1 | role=Z, 상태=RDS 기사 작성 페이지 진입 | 송고·보류·KILL **3개 버튼이 모두** 렌더된다 (D와 달리 KILL도 보임) |
| 2 | role=Z 이지만 상태가 RDS가 아닌 기사(예: DPS)를 편집 진입 | 송고·보류·KILL **어느 버튼도 노출되지 않는다** (RDS gate) |
| 3 | 제목 채움 + 본문에 "(끝)" 없음 → 송고 클릭 후 confirm "확인" | `window.alert('본문에 (끝) 표시가 없어 송고할 수 없습니다.')`가 호출되고, `saveArticle`/`applyAction`(서버 전송)이 **한 번도 호출되지 않으며** 페이지가 초기화되지 않는다 |
| 4 | 제목 채움 + 본문 끝에 "(끝)"(Alt+Y) 있음 → 송고 클릭 후 confirm "확인" | ALERT 없이 정상 진행되어 `saveArticle` 후 `applyAction(..., 'send')`이 호출되고, 성공 시 페이지가 초기화된다 |
| 5 | 본문에 "(끝)" 없음 → **보류** 또는 **KILL** 클릭 후 confirm "확인" | (끝) ALERT가 **뜨지 않고** 정상 진행된다 — (끝) 가드는 송고에만 적용됨을 확인 (보류는 제목 가드는 적용, KILL은 제목 가드도 면제) |

### 보조 검증 포인트
- (#3) 송고 클릭 시 `window.confirm`에서 **취소**하면 ALERT조차 뜨지 않고 어떤 저장/전송도 없어야 한다.
- (#4) "(끝)"은 본문 **맨 끝**에 있어야 통과한다 — 본문 중간에만 "(끝)"이 있고 뒤에 다른 텍스트가 이어지면 `hasEndMarker`가 false라 차단되어야 한다.
- (#1/#2) 버튼 노출은 `user.role`과 `ctrl.status` 두 조건의 AND로만 결정되며, status가 RDS가 아니면 권한과 무관하게 전부 숨겨진다.
